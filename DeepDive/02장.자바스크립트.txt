2.1 자바스크립트의 탄생

    1995년, 웹페이지의 보조적인 기능을 수행하기 위해 브라우저에서 동작하는 경량 프로그래밍 언어로 개발된 것이 시초

2.2 자바스크립트의 표준화

    1997년 7월 자바스크립트의 표준화 이슈와 함께 ECMAScript로 이름이 바뀐다

2.3 자바스크립트 성장의 역사

    초창기에는 웹페이지의 보조적인 기능을 수행하는 한정적인 용도로 사용되었다

    (1) Ajax
        자바스크립트를 이용해 서버와 브라우저가 비동기 방식으로 데이터를 교환할 수 있는 통신 기능
        Ajax를 통해 서버로부터 필요한 데이터만 전송받아 렌더링하여 성능을 상승시켰다

    (2) jQuery
        2006년 당시 논란이 있던 DOM을 더욱 쉽게 제어 가능, 배우기 쉽고 직관적인 특징을 지닌다

    (3) V8 자바스크립트 엔진
        2008년에 등장하여 데스크톱 애플리케이션과 유사한 UX를 제공, 빠른 성능

    (4) Node.js
        v8 엔진으로 빌드된 자바스크립트 런타임 환경으로 브라우저 이외의 환경에서도 동작
        비동기 I/O를 지원하며 단일 스레드 이벤트 루프 기반으로 동작

    (5) SPA 프레임워크
        CBD 방법론을 기반으로 하는 SPA가 대중화

2.4 자바스크립트와 ECMAScript

    ECMAScript가 기본 뼈대이며 브라우저가 별도 지원하는 클라이언트 사이드 Web API가 합쳐져 자바스크립트가 동작

2.5 자바스크립트의 특징

    HTML, CSS와 함께 웹을 구성하는 요소 중 하나
    인터프리터 언어로 컴파일러 언어에 비해 처리 속도가 느리지만 동적 기능이 지원된다
    멀티 패러다임 프로그래밍 언어 (명령형, 함수형, 프로토타입 기반 객체지향 프로그램밍)

2.6 ES6 브라우저 지원 현황

    인터넷 익스플로러를 제외한 모던 브라우저의 ES6 지원 비율은 거의 100%에 육박한다
