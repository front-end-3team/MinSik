22.1 this 키워드

    메서드가 자신이 속한 객체의 프로퍼티를 참조하고 변경하려면, 먼저 객체를 가리키는 식별자를 참조해야 한다
    객체 리터럴을 통해 재귀적으로 참조하는 방식보단, 생성자 함수와 new 연산자를 이용하는 경우 이후 생성될 인스턴스를 가리키는 식별자가 필요하다
    => this는 자신이 속한 객체나 생성할 인스턴스를 가리키는 자기 참조 변수이며, this를 통해 참조한다
    => this는 함수가 호출되는 방식에 따라 동적으로 바인딩이 결정된다
    => 일반적으로 객체의 메서드 내부 또는 생성자 함수 내부에서만 의미가 있다

22.2 함수 호출 방식과 this 바인딩

    (1) 일반 함수 호출
        기본적으로 this에는 전역 객체가 바인딩된다
        => 어떤 함수라도 일반 함수로 호출되면 this에 전역 객체가 바인딩된다

    (2) 메서드 호출
        메서드를 호출한 객체에 바인딩되는 것이지 메서드를 소유한 객체가 아니다!

    (3) 생성자 함수 호출
        생성자 함수가 미래에 생성할 인스턴스가 바인딩된다

    (4) Function.prototype.apply/call/bind 메서드에 의한 간접 호출
        apply, call은 호출할 함수에 인수를 전달한다
        bind는 인수로 this 바인딩이 교체된 함수를 생성한다
