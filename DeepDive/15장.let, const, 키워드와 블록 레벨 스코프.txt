15.1 var 키워드로 선언한 변수의 문제점

    (1) 변수 중복 선언 허용
        var 키워드는 변수의 중복을 허용한다

    (2) 함수 레벨 스코프
        var 키워드는 오직 함수 코드 블록만 지역 스코프로 인정
        => 함수 블록을 제외하면 모두 전역 변수

    (3) 변수 호이스팅
        가독성 ↓, 오류 발생 가능

15.2 let 키워드

    (1) 변수 중복 선언 금지
        이름이 중복되면 오류 발생

    (2) 블록 레벨 스코프   
        모든 코드 블록을 지역 스코프로 인정

    (3) 변수 호이스팅
        선언과 초기화가 분리되어 진행
        => 호이스팅이 발생하지 않는 것처럼 동작

    (4) 전역 객체와 let
        let 키워드로 선언한 전역 변수는 전역 객체의 프로퍼티가 아니다

15.3 const 키워드는

    (1) 선언과 초기화
        반드시 선언과 동시에 초기화해야 한다

    (2) 재할당 금지

    (3) 상수
        재할당이 금지된 변수
        => 상태 유지, 가독성, 유지보수의 편의를 위해 적극 사용

    (4) const 키워드와 객체
        재할당만 금지할 뿐, 불변은 아니다!!
        => 객체값이 할당된다면 값을 변경할 수 있다

15.4 var vs. let vs. const

    재할당이 필요한 경우, let을 사용
    => 보통의 경우 const 주로 사용하자
