* 조건문은 코드의 실행 흐름을 변경하므로 가독성을 해친다
  => 그러므로, forEach, map, filter, reduce 같은 고차 함수 사용을 권장한다

8.1 블록문

    블록문 자체가 ;과 같은 의미 => 자체 종결성
    블록문 === 하나의 실행 단위

8.2 조건문

    불리언 값으로 평가될 수 있는 표현식

    (1) if ... else 문
        if문의 조건식이 불리언 값이 아니어도 암묵적으로 강제 변환된다!
        조건이 간단하다면 삼항 조건 연산자가 가독성이 훨 좋다

    (2) switch 문
        논리적 참/거짓 보단 다양한 상황에 따라 실행할 코드 블록을 결정할 때 사용
        default를 제외하면 break문이 별도로 필요하다

8.3 반복문

    forEach, for in, for of 문으로 대체 가능

    (1) for 문
        for 문 내에 for 문을 중첩한 이중 for 문이 존재
        무한루프인 경우, for ( ;; ) { ... }

    (2) while 문
        조건식이 참이면 무한루프인
    
    (3) do ... while 문
        무조건 한 번은 실행

8.4 break 문

    레이블 문, 반복문, switch 문의 코드 블록에서 사용하면 탈출

8.5 continue 문

    코드 블록 실행을 현 지점에서 중단하고 반복문의 증감식으로 실행 흐름을 이동시킨다
