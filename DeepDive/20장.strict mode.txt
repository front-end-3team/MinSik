20.1 strict mode란?

    자바스크립트 엔진은 전역 객체에 프로퍼티를 동적으로 생성하여 전역 변수처럼 참조하는 암묵적 전역 현상이 있다
    => 이와 같은 문제를 해결하기 위해 자바스크립트 문법을 좀 더 엄격히 적용시키는 strict mode(엄격 모드)가 추가되었다
    => ESLint 도구를 사용해도 같은 효과를 볼 수 있다
    => ES6에서 도입된 클래스와 모듈은 기본적으로 strict mode가 적용된다

20.2 strict mode의 적용

    'use strict';을 추가하면 적용시킬 수 있다
    => 코드의 선두에 위치시키지 않는다면 제대로 동작하지 않는다

20.3 전역에 strict mode를 적용하는 것은 피하자

    strict mode는 스크립트 단위로 적용된다
    => 때문에 다른 스크립트와 혼용된다면 오류를 발생시킬 수 있다
    => 즉시 실행 함수로 감싸고 적용하자

20.4 함수 단위로 strict mode를 적용하는 것도 피하자

    즉시 실행 함수로 감싸고 적용하자

20.5 stirct mode가 발생시키는 에러

    (1) 암묵적 전역
        선언하지 않은 변수를 참조하면 ReferenceError가 발생한다

    (2) 변수, 함수, 매개변수의 삭제
        delete 연산자로 삭제하면 SyntaxError가 발생한다

    (3) 매개변수 이름의 중복
        SyntaxError가 발생한다

    (4) with 문의 사용
        SyntaxError가 발생한다
        => with 문은 전달된 객체를 스코프 체인에 추가하여 동일한 프로퍼티를 반복 시 객체 이름을 생략할 수 있다
        => 코드가 간단해지긴 하지만 성능과 가독성이 나빠지므로 사용하지 말자
